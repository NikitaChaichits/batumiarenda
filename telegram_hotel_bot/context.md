# Контекст проекта: Telegram-бот отеля / апартаментов (Batumi)

Документ для быстрой ориентации в коде и договорённостях. **После крупных изменений** (новый раздел меню, схема БД, деплой, интеграции) этот файл стоит дополнять краткой записью в разделе «Журнал».

---

## Назначение

Гостевой Telegram-бот для отеля/апартаментов в Батуми: язык RU/EN, каталог объявлений (сейчас из Airbnb), FAQ, отзывы (с модерацией), полезные ссылки на карты, поддержка с эскалацией админам, заглушка лояльности.

---

## Стек

| Компонент | Выбор |
|-----------|--------|
| Язык | Python 3.x |
| Bot API | **aiogram 3.x** (async, long polling) |
| БД | **SQLite** + `aiosqlite` |
| FSM | `MemoryStorage` (состояние сбрасывается при перезапуске процесса) |
| Секреты | `python-dotenv`, файл `.env` рядом с `config.py` |
| Зависимости | `requirements.txt` |

## Структура репозитория (папка `telegram_hotel_bot/`)

| Путь | Роль |
|------|------|
| `main.py` | Точка входа: `Dispatcher`, роутеры, `init_db` в `startup` |
| `config.py` | `BOT_TOKEN`, `ADMIN_IDS`, `BOOKING_BASE_URL`, лимиты отзывов/поддержки |
| `database.py` | Схема SQLite, `AIRBNB_LISTINGS`, миграции по `_CATALOG_DB_VERSION`, `fetch_recent_reviews` |

| `handlers/` | Роутеры: `start`, `rooms`, `faq`, `reviews`, `support`, `useful_links`, `loyalty` |
| `keyboards/` | Reply и inline; `review_menu`, `useful_links` |
| `translations/` | `ru.py`, `en.py`, общий `t()` и FAQ |
| `hotel_bot.db` | SQLite (не коммитить при необходимости — в `.gitignore` по желанию) |
| `.env` | Секреты (в `.gitignore`) |

## Запуск

```bash
cd telegram_hotel_bot
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

Рабочая директория должна позволять импорт `handlers`, `keyboards`, `translations` как пакетов при запуске `main.py` из этой папки.

## Конфигурация (`.env`)

| Переменная | Описание |
|------------|----------|
| `BOT_TOKEN` | Обязательно. Токен от @BotFather |
| `ADMIN_IDS` | Необязательно. Telegram **user id** через запятую: поддержка, «уточнить даты», **модерация отзывов** (`rev:approve`) |
| `BOOKING_BASE_URL` | База для кнопки «Проверить даты на Airbnb», если у номера нет своего `listing_url` |
| `REVIEW_COOLDOWN_SECONDS` | Интервал между отзывами одного пользователя (по умолчанию 3600) |
| `SUPPORT_MAX_MESSAGES_PER_WINDOW` | Лимит сообщений админам за окно: **поддержка** и **уточнение дат** (по умолчанию 5) |
| `SUPPORT_RATE_WINDOW_SECONDS` | Размер окна в секундах (по умолчанию 3600) |

## База данных

- **Пользователи** — `telegram_id`, язык, имя/username.
- **Отзывы** — оценка, текст, флаг **`approved`** (1 — в ленте «Посмотреть»); у старых БД после миграции все существующие строки помечаются одобренными. Поле `photo_file_id` сохранено для старых записей.
- **Номера (`rooms`)** — заголовки RU/EN, описания, `price_per_night` (в БД; в подписи к фото не показывается), `image_url`, `images_json`, `listing_url`.
- **Rate events** — события для лимита поддержки.

Каталог из объявлений пересобирается при старте, если `PRAGMA user_version` ниже **`_CATALOG_DB_VERSION`** в `database.py`, или каталог пуст. Ручная перезаливка: `await database.reload_airbnb_catalog()` (например из одноразового скрипта).

Источник данных каталога: **`AIRBNB_LISTINGS`** (собирается из `_AIRBNB_PAIRS` + `_DESCRIPTIONS_RU` / `_DESCRIPTIONS_EN`).

## Поведение бота (ключевые сценарии)

1. **`/start`** — выбор языка (inline), затем приветствие и Reply-меню.
2. **Меню (Reply)** — **Номера · Все номера**, **Отзывы · FAQ**, **Поддержка · Язык**, отдельная строка **«Полезные ссылки»** (`main_reply_keyboard`).
3. **«Номера»** — карточки по одному лоту, листание «Назад/Вперёд», опционально галерея фото; кнопка **«Проверить даты на Airbnb»** (URL на объявление или `BOOKING_BASE_URL`); кнопка **«Уточнить даты у менеджера»** (`room:ask_dates:{index}`) → FSM: пользователь одним сообщением присылает даты → админам уходит текст вида «Пользователь — … / Апартаменты — … / хочет уточнить… — …» (тот же rate limit, что и у поддержки).
4. **«Все номера»** — заголовок с количеством, затем подряд карточки: **фото (первое из галереи)** + подпись (заголовок и описание), под каждой — **«Проверить даты на Airbnb»** и **«Уточнить даты у менеджера»** (см. `room_summary_actions_keyboard`).
5. **Колбэки карточки** — `room:idx:{index}:{photo}`; «край» списка — `room:block`. Устаревший `room:open:{index}` (список с №) по-прежнему открывает полную карточку, если остались старые сообщения.
6. **FAQ** — вопросы из `translations`; ответы в `FAQ_ANSWERS` или заглушка.
7. **Отзывы** — кнопка «Отзывы» → inline **«Оставить отзыв»** / **«Посмотреть отзывы»**. Запись: оценка 1–5 → текст. Если в `.env` задан **`ADMIN_IDS`**, новый отзыв сохраняется с `approved=0`, гость видит `review_thanks_pending`, каждому админу уходит карточка с **`rev:approve:{id}`**; в ленте попадают только `approved=1`. Если админов нет — сразу `approved=1` и короткое «спасибо». Просмотр: `fetch_recent_reviews` (только одобренные).
8. **«Полезные ссылки»** — текст + inline URL на Google Maps: локация отеля, Nikora / Carrefour / GoodWill, Metro City / Grand Mall (`keyboards/useful_links.py`).
9. **Поддержка** — `/support` или кнопка; одно текстовое сообщение админам; rate limit; `support_send_failed` если ни одному админу не ушло.
10. **`/menu`** — сброс FSM и возврат к главному меню.
11. **`/cancel`** — сброс FSM.
12. **`/loyalty`** — заглушка.

- Строки интерфейса: `translations/ru.py`, `translations/en.py` (`MESSAGES`, FAQ).

## Вспомогательно: фото с Airbnb

Модуль **`airbnb_photos.py`**: `fetch_listing_photo_urls(url)`, `merge_fetched_images(item)`. Запуск:

```bash
python -m airbnb_photos "https://www.airbnb.ru/rooms/..."
```

На некоторых macOS при ошибке SSL используется fallback без проверки сертификата (см. код). Парсинг HTML неофициальный — может устареть.

## Не забыть

- Админы должны хотя раз написать боту **`/start`**, иначе доставка в поддержку может не сработать.
- Не коммитить `.env` и при необходимости `hotel_bot.db`.
- Увеличивать **`_CATALOG_DB_VERSION`** (или вызывать `reload_airbnb_catalog()`), если меняли только код каталога без смены версии в уже развёрнутой БД.
